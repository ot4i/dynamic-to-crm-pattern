Constraints on using the Microsoft Dynamics CRM Account Entity output: Dynamic transport input pattern

A number of constraints apply when you use this pattern.

You can use the Microsoft Dynamics CRM Account Entity output: Dynamic transport input pattern only if you meet all of the following criteria:

  • The pattern can only be deployed and run on brokers in Microsoft Windows environments, because the .NETCompute node is dependent on the .NET Framework for connectivity to Microsoft Dynamics CRM Systems.
  • The message flow instance and BAR file contain the Microsoft Dynamics CRM connection credentials in clear text. If this is not suitable for your security requirements, you must modify the message flow so that the .NETCompute nodes can obtain these details by a secure mechanism. For more information, see Tasks to complete before applying the Microsoft Dynamics CRM Account Entity output: Dynamic transport input pattern.
  • The message flow instance can set and get only the standard attributes on the Microsoft Account Entity. You can modify and extend the flow instance created by the pattern for custom attributes. For information, see Tasks to complete before applying the Microsoft Dynamics CRM Account Entity output: Dynamic transport input pattern.

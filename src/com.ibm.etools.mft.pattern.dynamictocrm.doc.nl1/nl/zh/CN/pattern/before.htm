应用 Microsoft Dynamics CRM Account Entity output: Dynamic transport input 模式之前要完成的任务

在应用该模式以生成模式实例之前，请完成下列任务：

  • 确保此模式是否符合您的业务需求，以及请求者应用程序和提供者应用程序是否都支持此模式。
  • 选择模式实例的名称。识别模式应用程序的用途，以便选择合适的名称。
  • 确定要使用哪种传输方法，并准备传输详细信息配置。例如，创建 WebSphere MQ 传输方法所需的消息队列、为文件传输方法创建输入文件目录，或者为 HTTP 传输方法选择 URL 后缀。
  • 准备每项操作的输入和输出模式。
  • 准备必需的 Microsoft Dynamics CRM 系统连接信息。这包括：
    • CRMDomain
    • CRMURI
    • CRMUser
    • CRMUser 密码
  • 确定是否需要记录和错误消息。如果需要其中任一消息，您必须确定要向哪个队列管理器和队列发送消息。您可以将消息发送到由多个消息流使用的常规队列，也可以使用特定于模式实例的队列。

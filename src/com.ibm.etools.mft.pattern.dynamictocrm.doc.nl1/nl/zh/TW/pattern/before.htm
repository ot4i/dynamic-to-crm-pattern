套用 Microsoft Dynamics CRM Account Entity 輸出：動態傳輸輸入型樣之前要完成的作業

在套用型樣以產生型樣實例之前，完成下列作業：

  • 確保此型樣符合商業需求，而且要求者及提供者應用程式都可以支援此型樣。
  • 選擇型樣實例的名稱。識別型樣應用程式的用途，讓您可以選擇適當的名稱。
  • 決定您要使用的傳輸，並準備傳輸明細配置。例如，建立 WebSphere MQ 傳輸的必要訊息佇列，建立檔案傳輸的輸入檔目錄，或者選擇 HTTP 傳輸的 URL 字尾。
  • 準備每一個作業的輸入和輸出綱目。
  • 準備必要的 Microsoft Dynamics CRM 系統連線資訊。這包括：
    • CRMDomain
    • CRMURI
    • CRMUser
    • CRMUser 密碼
  • 決定您是否需要記載訊息和錯誤訊息。如果您需要其中任一訊息，則必須決定傳送訊息的目標佇列管理程式和佇列。您可以將訊息傳送至多個訊息流程所使用的一般佇列，也可以使用特別針對型樣實例的佇列。

Tâches à exécuter avant d'appliquer le modèle Entité de compte Microsoft Dynamics CRM en sortie : transport dynamique en entrée

Avant d'appliquer le modèle permettant de produire une instance du modèle, exécutez les tâches suivantes :

  • Vérifiez que ce modèle répond aux besoins de l'entreprise et qu'il peut être pris en charge par les applications demandeur et fournisseur.
  • Attribuez un nom à l'instance de modèle. Identifiez la fonction de l'application de modèle afin de choisir un nom approprié.
  • Décidez quel transport utiliser et préparez les configurations détaillées de transport. Par exemple, créez les files d'attente de messages requises pour le transport WebSphere MQ, créez des répertoires de fichier d'entrée pour le transport de fichiers ou choisissez le suffixe d'URL pour le transport HTTP.
  • Préparez les schémas d'entrée et de sortie pour chaque opération.
  • Préparez les informations de connexion du système Microsoft Dynamics CRM. Parmi lesquelles :
    • CRMDomain
    • CRMURI
    • CRMUser
    • Mot de passe CRMUser
  • Déterminez si des messages de journal et des messages d'erreur sont requis. Si vous avez besoin de l'un ou l'autre, vous devez identifier le gestionnaire de files d'attente et la file d'attente auxquels les messages seront envoyés. Vous pouvez envoyer ces derniers à une file d'attente générale utilisée par plusieurs flux de messages ou utiliser une file d'attente spécialement dédiée à l'instance du modèle.

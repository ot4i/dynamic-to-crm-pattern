Contraintes d'utilisation du modèle Entité de compte Microsoft Dynamics CRM en sortie : transport dynamique en entrée

Il existe certaines contraintes lorsque vous utilisez ce modèle.

Vous ne pouvez utiliser le modèle Entité de compte Microsoft Dynamics CRM en sortie : transport dynamique en entrée que si tous les critères suivants sont remplis :

  • Le modèle ne peut être déployé et exécuté que dans des environnements Microsoft Windows car le noeud .NETCompute dépend de .NET Framework pour la connectivité aux systèmes Microsoft Dynamics CRM.
  • L'instance de flux de messages et le fichier BAR contiennent les données d'identification de connexion Microsoft Dynamics CRM au format text en clair. Si cela n'est pas conforme à vos exigences de sécurité, modifiez le flux de messages afin que les noeuds .NETCompute puissent obtenir ces informations par le biais d'un mécanisme sécurisé. Pour plus d'informations, voir Tâches à exécuter avant d'appliquer le modèle Entité de compte Microsoft Dynamics CRM en sortie : transport dynamique en entrée.
  • L'instance de flux de messages peut définir et obtenir uniquement les attributs standard dans l'entité de compte Microsoft. Vous pouvez modifier et développer l'instance de flux créée par le modèle pour les attributs personnalisés. Pour plus d'informations, voir Tâches à exécuter avant d'appliquer le modèle Entité de compte Microsoft Dynamics CRM en sortie : transport dynamique en entrée.

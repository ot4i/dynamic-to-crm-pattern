Paramètres du modèle Entité de compte Microsoft Dynamics CRM en sortie : transport dynamique en entrée

Sélection d'opération
Paramètres du modèle Valeur par défaut Description
Opération

Choisissez l'une des opérations suivantes : création, extraction, mise à jour ou suppression. Des flux de messages connexes peuvent résulter de la génération du modèle. Vous devez sélectionner au moins une opération.
Modèle de transport en entrée
Paramètres du modèle Valeur par défaut Description
Transport en entrée

Choisissez l'un des trois transports suivants : MQ, File ou HTTP et configurez les informations relatives dans Détails du transport ci-dessous.
Informations de transport

Entrez les informations de transport en fonction du transport et de l'opération sélectionnés. Vous devez fournir le nom de la file d'attente d'entrée pour le transport MQ, le répertoire d'entrée pour le transport File et le suffixe URL pour le transport HTTP.
Configuration du schéma d'entrée utilisateur
Paramètres du modèle Valeur par défaut Description
Configuration du schéma

Indiquez le schéma de message d'entrée et de sortie et l'élément global pour l'opération sélectionnée.

L'élément global choisi pour l'entrée est utilisé comme objet d'entrée dans le fichier de mappe approprié. Par exemple, si vous choisissez l'opération de création et indiquez le schéma d'entrée et l'élément global, ce dernier est utilisé comme objet d'entrée d'un fichier "Create.map". Le schéma CRMCreate qui contient CRMCreate comme élément global, est utilisé comme objet de sortie.

L'élément global choisi pour la sortie est utilisé comme objet de sortie dans le fichier de retour de mappe approprié. Par exemple, si vous avez spécifié un schéma de sortie et un élément global pour l'opération de création, l'élément global est utilisé comme objet de sortie d'un fichier "CreateReturn.map". Le schéma CRMCreate qui contient CRMCreate comme élément global, est utilisé comme objet d'entrée.
Connexion CRM
Paramètres du modèle Valeur par défaut Description
URL d'organisation CRM

Ce paramètre de modèle définit l'URL de service de l'organisation du système CRM. Il doit s'agir de l'URL complète du service de l'organisation. Par exemple, pour accéder à CRM sur site, http://ipaddress:port/orgName/XRMServices/2011/Organization.svc ; pour accéder à l'aide en ligne CRM, https://orgName.crm.dynamics.com/XRMServices/2011/Organization.svc.
Nom d'utilisateur CRM

Ce paramètre de modèle décrit le nom d'utilisateur CRM que vous avez utilisé sur le système CRM.
Mot de passe d'utilisateur CRM

Ce paramètre de modèle définit le mot de passe de l'utilisateur CRM que vous avez spécifié.
Domaine CRM

Ce paramètre de modèle définit le domaine CRM que vous avez utilisé.
Consignation
Paramètres du modèle Valeur par défaut Description
Consignation Effacé Ce paramètre du modèle indique si l'instance du modèle contient le flux secondaire de consignation.

Si ce paramètre de modèle est effacé, le flux secondaire de consignation n'est pas inclus.

Si ce paramètre de modèle est sélectionné, vous devez définir des valeurs pour Gestionnaire de files d'attente de journaux et File d'attente de consignation.
Gestionnaire de files d'attente de journaux Ce paramètre de modèle définit le gestionnaire de files d'attente utilisé pour la consignation. Vous pouvez laisser ce paramètre à blanc si le gestionnaire de files d'attente du courtier est utilisé pour la consignation.

Ce paramètre de modèle n'est requis que si Consignation nécessaire est sélectionné.
File d'attente de consignation Ce paramètre de modèle définit la file d'attente utilisée pour la consignation.

Ce paramètre de modèle n'est requis que si Consignation nécessaire est sélectionné.
Traitement d'erreurs
Paramètres du modèle Valeur par défaut Description
Erreur de consignation Sélectionné Ce paramètre de modèle indique si l'instance du modèle contient le code d'insertion des message d'erreur dans une file d'attente.

Si ce paramètre de modèle est sélectionné, vous devez définir les valeurs de Gestionnaire de files d'attente d'erreurs et File d'attente d'erreurs.
Gestionnaire de files d'attente d'erreurs Ce paramètre de modèle définit le gestionnaire de files d'attente utilisé pour la consignation des erreurs.

Vous devez le définir uniquement si vous sélectionnez Message d'erreur nécessaire.
File d'attente d'erreurs Ce paramètre de modèle définit la file d'attente utilisée pour les messages d'erreur.

Vous devez le définir uniquement si vous sélectionnez Message d'erreur nécessaire.

Tous les paramètres du modèle sont répertoriés avec leur valeur par défaut. Lorsqu'aucune valeur n'est définie, la valeur par défaut est une chaîne vide.

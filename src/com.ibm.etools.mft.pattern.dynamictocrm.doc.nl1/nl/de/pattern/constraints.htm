Einschränkungen bei Verwendung des Musters 'Microsoft Dynamics CRM Account Entity output: Dynamic transport input'

Bei Verwendung dieses Musters gelten eine Reihe von Einschränkungen.

Das Muster 'Microsoft Dynamics CRM Account Entity output: Dynamic transport input' kann nur verwendet werden, wenn die folgenden Bedingungen alle erfüllt sind:

  • Das Muster kann nur auf Brokern in Microsoft Windows-Umgebungen implementiert und ausgeführt werden, weil der .NETCompute-Knoten für Verbindungen mit Microsoft Dynamics CRM-Systemen von .NET Framework abhängig ist.
  • Die Nachrichtenflussinstanz und die BAR-Datei enthalten die Berechtigungsnachweise für die Microsoft Dynamics CRM-Verbindung in Klartext. Falls dies gegen Ihre Sicherheitsanforderungen verstößt, müssen Sie den Nachrichtenfluss so ändern, dass die .NETCompute-Knoten diese Details in einem sicheren Verfahren abrufen. Weitere Informationen finden Sie im Abschnitt Erforderliche Tasks vor der Anwendung des Musters 'Microsoft Dynamics CRM Account Entity output: Dynamic transport input'.
  • Die Nachrichtenflussinstanz kann nur die Standardattribute für die Microsoft-Accountentität festlegen und abrufen. Sie können die vom Muster erstellte Flussinstanz für angepasste Attribute ändern und erweitern. Weitere Informationen finden Sie im Abschnitt Erforderliche Tasks vor der Anwendung des Musters 'Microsoft Dynamics CRM Account Entity output: Dynamic transport input'.

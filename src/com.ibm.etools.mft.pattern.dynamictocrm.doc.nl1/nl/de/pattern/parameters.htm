Parameter für das Muster Microsoft Dynamics CRM Account Entity output: Dynamic transport input

Operationsauswahl
Musterparameter Standardwert Beschreibung
Operation

Wählen Sie eine oder mehrere der Operationen Erstellen, Abrufen, Aktualisieren und Löschen aus. Nach der Generierung des Musters können Sie über entsprechende Nachrichtenflüsse verfügen. Sie müssen mindestens eine Operation auswählen.
Muster Eingabetransport
Musterparameter Standardwert Beschreibung
Eingabetransport

Wählen Sie eine der drei Transportmethoden (MQ, Datei und HTTP) aus und konfigurieren Sie unten in 'Transportdetails' entsprechende Details.
Transportdetails

Eingabetransportdetails auf Basis der Transportmethode und der Operation, die Sie ausgewählt haben. Sie müssen für MQ-Transport den Namen der Eingabewarteschlange, für Dateitransport ein Eingabeverzeichnis und für HTTP-Transport das URL-Suffix angeben.
Benutzereingabe Schemakonfiguration
Musterparameter Standard Beschreibung
Schemakonfiguration

Geben Sie das Eingabe- und Ausgabenachrichtenschema und das globale Element für die ausgewählte Operation an.

Das für die Eingabe ausgewählte globale Element wird als Eingabeobjekt in der relevanten Zuordnungsdatei verwendet. Wenn Sie beispielsweise die Erstellungsoperation auswählen und das Eingabeschema und das globale Element angeben, wird das globale Element als Eingabeobjekt einer "Create.map"-Datei verwendet. Das CRMCreate-Schema, das CRMCreate als sein globales Element enthält, wird als Ausgabeobjekt verwendet.

Das für die Ausgabe ausgewählte globale Element wird als Ausgabeobjekt in der relevanten Zuordnungsrückgabedatei verwendet. Wenn Sie beispielsweise ein Ausgabeschema und ein globales Element für die Erstellungsoperation angegeben haben, wird das globale Element als Ausgabeobjekt einer "CreateReturn.map"-Datei verwendet. Das CRMCreate-Schema, das CRMCreate als sein globales Element enthält, wird als Eingabeobjekt verwendet.
CRM-Verbindung
Musterparameter Standardwert Beschreibung
CRM-Unternehmens-URL

Dieser Musterparameter definiert die Unternehmensservice-URL des CRM-Systems. Es muss eine vollständige URL des Unternehmensservice angegeben werden. Beispiele: Beim Zugriff auf CRM vor Ort http://ipaddress:port/orgName/XRMServices/2011/Organization.svc und beim Onlinezugriff auf CRM https://orgName.crm.dynamics.com/XRMServices/2011/Organization.svc.
CRM-Benutzername

Dieser Musterparameter gibt den CRM-Benutzernamen an, den Sie im CRM-System verwendet haben.
CRM-Benutzerkennwort

Dieser Musterparameter gibt das Kennwort für den angegebenen CRM-Benutzer an.
CRM-Domäne

Dieser Musterparameter gibt die CRM-Domäne an, die Sie verwendet haben.
Protokollierung
Musterparameter Standard Beispiel
Protokollierung ein Nicht ausgewählt Dieser Musterparameter bestimmt, ob die Musterinstanz den untergeordneten Nachrichtenfluss für die Protokollierung enthält.

Wenn dieser Musterparameter gelöscht wird, ist der Protokollierungs-Unterablauf nicht enthalten.

Wenn dieser Musterparameter ausgewählt ist, müssen Sie Werte für Protokoll-WS-Manager und Protokollwarteschlange festlegen.
Protokoll-WS-Manager Dieser Musterparameter definiert den Warteschlangenmanager, der zur Protokollierung verwendet wird. Sie können diesen Parameter leer lassen, wenn der Warteschlangenmanager des Brokers für die Protokollierung verwendet wird.

Dieser Musterparameter ist nur erforderlich, wenn Protokollierung erforderlich ausgewählt ist.
Protokollwarteschlange Dieser Musterparameter definiert die Warteschlange, die zur Protokollierung verwendet wird.

Dieser Musterparameter ist nur erforderlich, wenn Protokollierung erforderlich ausgewählt ist.
Fehlerbehandlung
Musterparameter Standard Beispiel
Fehlerprotokollierung ein Ausgewählt Dieser Musterparameter bestimmt, ob die Musterinstanz Code für das Einreihen von Fehlernachrichten in eine Warteschlange enthält.

Wenn dieser Musterparameter aktiviert ist, müssen für Fehler-WS-Manager und Fehlerwarteschlange Werte eingegeben werden.
Fehler-WS-Manager Dieser Musterparameter definiert den Warteschlangenmanager, der zur Fehlerprotokollierung verwendet wird.

Dieser Musterparameter ist nur erforderlich, wenn Fehlernachricht erforderlich ausgewählt ist.
Fehlerwarteschlange Dieser Musterparameter definiert die Warteschlange, die für Fehlernachrichten verwendet wird.

Dieser Musterparameter ist nur erforderlich, wenn Fehlernachricht erforderlich ausgewählt ist.

Zu allen Musterparametern sind die Standardwerte angegeben. Wenn kein Standardwert angegeben ist, ist der Standardwert eine leere Zeichenfolge.

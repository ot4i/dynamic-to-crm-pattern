Erforderliche Tasks vor der Anwendung des Musters 'Microsoft Dynamics CRM Account Entity output: Dynamic transport input'

Bevor Sie das Muster anwenden, um eine Musterinstanz zu erstellen, müssen Sie folgende Tasks ausführen:

  • Stellen Sie sicher, dass dieses Muster Ihren Geschäftsanforderungen entspricht und sowohl von den Requester- als auch von den Provideranwendungen unterstützt wird.
  • Wählen Sie einen Namen für die Musterinstanz aus. Bestimmen Sie den Zweck der Musteranwendung, sodass Sie einen geeigneten Namen auswählen können.
  • Entscheiden Sie, welche Transportmethode Sie verwenden möchten und bereiten Sie die Detailkonfigurationen für den Transport vor. Erstellen Sie zum Beispiel die erforderlichen Nachrichtenwarteschlangen für den WebSphere MQ-Transport, erstellen Sie Eingabedateiverzeichnisse für den Dateitransport oder wählen Sie das URL-Suffix für den HTTP-Transport aus.
  • Bereiten Sie Eingabe- und Ausgabeschemas für jede Operation vor.
  • Bereiten Sie die erforderlichen Microsoft Dynamics CRM-Systemverbindungsinformationen vor. Dies schließt folgende Informationen ein:
    • CRMDomain
    • CRMURI
    • CRMUser
    • CRMUser-Kennwort
  • Entscheiden Sie, ob eine Protokollierung und Fehlernachrichten erforderlich sind. Wenn eins davon erforderlich ist, müssen Sie festlegen, an welchen WS-Manager und an welche Warteschlange die Nachrichten gesendet werden sollen. Die Nachrichten können an eine allgemeine Warteschlange gesendet werden, die von mehreren Nachrichtenflüssen verwendet wird, oder an eine speziell für die Musterinstanz eingerichtete Warteschlange.

Tareas que deben completarse antes de aplicar el patrón Salida de entidad de cuenta de Microsoft Dynamics CRM: entrada de transporte dinámica

Antes de aplicar el patrón para generar una instancia de patrón, realice las tareas siguientes:

  • Asegúrese de que este patrón cumple los requisitos empresariales y puede ser soportado por las aplicaciones de solicitante y de proveedor.
  • Elija un nombre para la instancia de patrón. Identifique la finalidad de la aplicación de patrón para que pueda elegir un nombre adecuado.
  • Decida qué transporte desea utilizar y prepare las configuraciones detalladas de transporte. Por ejemplo, cree las colas de mensajes necesarias para el transporte WebSphere MQ, cree directorios de archivos de entrada para el transporte de archivos o elija el sufijo de URL para el transporte HTTP.
  • Prepare esquemas de entrada y salida para cada operación.
  • Prepare la información de conexión necesaria del sistema Microsoft Dynamics CRM. Ésta incluye:
    • Dominio de CRM
    • URI de CRM
    • Usuario de CRM
    • Contraseña del usuario de CRMU
  • Decida si necesita mensajes de registro y de error. Si necesita cualquiera de ellos, debe decidir a qué gestor de colas y a qué cola se enviarán los mensajes. Puede enviar mensajes a una cola general que utilicen varios flujos de mensajes, o puede utilizar una cola específicamente para la instancia de patrón.

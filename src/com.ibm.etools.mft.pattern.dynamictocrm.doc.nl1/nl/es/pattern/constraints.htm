Restricciones en el uso del patrón Salida de entidad de cuenta de Microsoft Dynamics CRM: entrada de transporte dinámica

Se aplican una serie de restricciones cuando se utiliza este patrón.

Sólo puede utilizar el patrón Salida de entidad de cuenta de Microsoft Dynamics CRM: entrada de transporte dinámica si se cumplen todos los criterios siguientes:

  • El patrón sólo se puede desplegar y ejecutar en intermediarios de entornos Microsoft Windows, ya que el nodo .NETCompute depende de .NET Framework para la conectividad con los sistemas Microsoft Dynamics CRM.
  • La instancia de flujo de mensajes y el archivo BAR contienen las credenciales de conexión de Microsoft Dynamics CRM en texto simple. Si esto no es adecuado para sus requisitos de seguridad, debe modificar el flujo de mensajes para que los nodos .NETCompute puedan obtener estos detalles mediante un mecanismo de seguridad. Para obtener más información, consulte Tareas que deben completarse antes de aplicar el patrón Salida de entidad de cuenta de Microsoft Dynamics CRM: entrada de transporte dinámica.
  • La instancia de flujo de mensajes puede establecer y obtener únicamente los atributos estándar de la Entidad de cuenta Microsoft. Puede modificar y ampliar la instancia de flujo creada por el patrón para los atributos personalizados. Para obtener información, consulte Tareas que deben completarse antes de aplicar el patrón Salida de entidad de cuenta de Microsoft Dynamics CRM: entrada de transporte dinámica.

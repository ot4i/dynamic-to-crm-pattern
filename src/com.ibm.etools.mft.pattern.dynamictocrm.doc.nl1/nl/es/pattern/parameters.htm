Parámetros para el patrón Salida de entidad de cuenta de Microsoft Dynamics CRM: entrada de transporte dinámica

Selección de operación
Parámetro de patrón Valor predeterminado Descripción
Operación

Elija una o más de las siguientes operaciones: Crear, Recuperar, Actualizar y Suprimir. Puede tener flujos de mensajes relativos después de generar el patrón. Debe seleccionar al menos una operación.
Transporte de entrada del patrón
Parámetro de patrón Valor predeterminado Descripción
Transporte de entrada

Elija uno de los tres transportes, MQ, File o HTTP, y configure detalles relativos en Detalles del transporte más abajo.
Detalles del transporte

Detalles del transporte de entrada basados en el transporte y la operación que ha seleccionado. Debe proporcionar el nombre de cola de entrada para el transporte MQ, el directorio de entrada para el transporte File y el sufijo de URL para el transporte HTTP.
Configuración de esquema de entrada de usuario
Parámetro de patrón Valor predeterminado Descripción
Configuración de esquema

Especifique el esquema de mensaje de entrada y de salida y el elemento global para la operación seleccionada.

El elemento global que elija para entrada se utiliza como el objeto de entrada en el archivo de correlación pertinente. Por ejemplo, si elige la operación Crear y especifica el esquema de entrada y el elemento global, el elemento global se utiliza como el objeto de entrada de un archivo "Create.map". El esquema CRMCreate, que contiene CRMCreate como su elemento global, se utiliza como el objeto de salida.

El elemento global que elija para salida se utiliza como el objeto de salida en el archivo de devolución de correlación pertinente. Por ejemplo, si ha especificado un esquema de salida y un elemento global para la operación Crear, el elemento global se utiliza como el objeto de salida de un archivo "CreateReturn.map". El esquema CRMCreate, que contiene CRMCreate como su elemento global, se utiliza como el objeto de entrada.
Conexión CRM
Parámetro de patrón Valor predeterminado Descripción
URL de organización de CRM

Este parámetro de patrón define el URL de servicio de organización del sistema CRM. Debe ser un URL completo al servicio de organización. Por ejemplo, acceder a CRM in situ, http://dirección_ip:puerto/orgName/XRMServices/2011/Organization.svc; acceder a CRM en línea, https://nombre_org.crm.dynamics.com/XRMServices/2011/Organization.svc.
Nombre de usuario de CRM

Este parámetro de patrón describe el nombre de usuario de CRM que ha utilizado en el sistema CRM.

Contraseña de usuario de CRM

Este parámetro de patrón define la contraseña para el usuario de CRM que ha especificado.
Dominio de CRM

Este parámetro de patrón define el dominio de CRM que ha utilizado.
Registro
Parámetro de patrón Valor predeterminado Descripción
Registro activado No seleccionado Este parámetro de patrón determina si la instancia de patrón incluye el subflujo de registro.

Si no se selecciona este parámetro de patrón, el subflujo de registro no se incluye.

Si se selecciona este parámetro de patrón, debe establecer valores para Gestor de colas de registro y Cola de registro.
Gestor de colas de registro Este parámetro de patrón define el gestor de colas que se utiliza para el registro. Puede dejar este parámetro en blanco si se utiliza el gestor de colas de intermediario para el registro.

Este parámetro de patrón sólo es necesario si se selecciona Registro necesario.
Cola de registro Este parámetro de patrón define la cola que se utiliza para el registro.

Este parámetro de patrón sólo es necesario si se selecciona Registro necesario.
Manejo de errores
Parámetro de patrón Valor predeterminado Descripción
Registro de errores activado Seleccionado Este parámetro de patrón determina si la instancia de patrón incluye código para colocar mensajes de error en una cola.

Si se selecciona este parámetro de patrón, se deben establecer valores para Gestor de colas de errores y Cola de errores.
Gestor de colas de errores Este parámetro de patrón define el gestor de colas que se utiliza para el registro cronológico de errores.

Este parámetro de patrón sólo es necesario si se selecciona Mensaje de error necesario.
Cola de errores Este parámetro de patrón define la cola que se utiliza para los mensajes de error.

Este parámetro de patrón sólo es necesario si se selecciona Mensaje de error necesario.

Todos los parámetros de patrón aparecen en la lista con sus valores predeterminados. Cuando no se indique un valor predeterminado, éste es una serie vacía.

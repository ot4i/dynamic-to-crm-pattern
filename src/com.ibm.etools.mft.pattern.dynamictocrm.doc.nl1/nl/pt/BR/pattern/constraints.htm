Restrições no Uso do Padrão Saída de Entidade de Conta do Microsoft Dynamics CRM: Entrada de Transporte Dinâmico

Várias restrições se aplicam quando este padrão é usado.

Você só pode usar o padrão de Saída de Entidade de Conta do Microsoft Dynamics CRM: Entrada de Transporte Dinâmico se atender a todos os critérios a seguir:

  • O padrão só pode ser implementado e executado em brokers em ambientes do Microsoft Windows, pois o nó .NETCompute é dependente do .NET Framework para conectividade com os Sistemas Microsoft Dynamics CRM.
  • A instância de fluxo de mensagens e o arquivo BAR contêm as credenciais de conexão do Microsoft Dynamics CRM em texto não criptografado. Se isso não for adequado para os seus requisitos de segurança, você deve modificar o fluxo de mensagem de modo que os nós .NETCompute possam obter esses detalhes por um mecanismo de segurança. Para obter mais informações, consulte Tarefas a Serem Concluídas antes da Aplicação do Padrão de Saída da Entidade de Conta do Microsoft Dynamics CRM: Entrada de Transporte Dinâmico.
  • A instância de fluxo de mensagens pode configurar e obter apenas os atributos padrão do Microsoft Account Entity. É possível modificar e estender a instância do fluxo criado pelo padrão para atributos customizados. Para obter informações, consulte Tarefas a Serem Concluídas antes da Aplicação do Padrão de Saída da Entidade de Conta do Microsoft Dynamics CRM: Entrada de Transporte Dinâmico.

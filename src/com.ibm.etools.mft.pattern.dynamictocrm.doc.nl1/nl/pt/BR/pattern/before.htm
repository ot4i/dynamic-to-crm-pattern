Tarefas a Serem Concluídas antes da Aplicação do Padrão de Saída da Entidade de Conta do Microsoft Dynamics CRM: Entrada de Transporte Dinâmico

Antes de aplicar o padrão para produzir uma instância padrão, conclua as seguintes tarefas:

  • Assegure que esse padrão satisfaça seus requisitos de negócios e pode ser suportado por aplicativos do solicitante e do fornecedor.
  • Escolha um nome para a instância padrão. Identifique o propósito do aplicativo padrão para que seja possível escolher um nome apropriado.
  • Decida que transporte você deseja usar, e prepare as configurações de detalhe do transporte. Por exemplo, crie as filas de mensagem necessárias para o transporte do WebSphere MQ, crie os diretórios de arquivo de entrada para o transporte de arquivos, ou escolha o sufixo da URL para o transporte de HTTP.
  • Prepare os esquemas de entrada e saída para cada operação.
  • Prepare as informações de conexão de sistema necessárias do Microsoft Dynamics CRM. Isso inclui:
    • CRMDomain
    • CRMURI
    • CRMUser
    • Senha do CRMUser
  • Decida se você necessita de mensagens de login e de erro. Se precisar de tais mensagens, você deve decidir a que gerenciador de fila e fila deve enviar as mensagens. É possível enviar mensagens a uma fila geral que é usada por diversos fluxos de mensagens ou é possível usar uma fila especificamente para a instância padrão.

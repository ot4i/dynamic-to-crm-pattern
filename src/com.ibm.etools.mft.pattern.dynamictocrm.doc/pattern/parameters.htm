Parameters for the Microsoft Dynamics CRM Account Entity output: Dynamic transport input pattern

Operation Selection
Pattern parameter Default Description
Operation

Choose one or more operations from Create, Retrieve, Update and Delete. You can have relative message flows after generating the pattern. You must select at least one operation.
Pattern Input Transport
Pattern parameter Default Description
Input Transport

Choose one of the three transports, MQ, File, and HTTP, and configure relative details in Transport Details below.
Transport Details

Input transport details based on the transport and operation you selected. You must provide the input queue name for MQ transport, input directory for File transport, and the URL suffix for HTTP transport.
User Input Schema Configuration
Pattern parameter Default Description
Schema Configuration

Specify the input and output message schema and global element for the selected operation.

The global element you chose for input is used as the input object in the relevant map file. For example, if you choose the Create operation and specify the input schema and global element, the global element is used as the input object of a "Create.map" file. The CRMCreate schema, containing CRMCreate as its global element, is used as the output object.

The global element you chose for output is used as the output object in the relevant map return file. For example, if you specified a output schema and global element for the Create operation, then the global element is used as the output object of a "CreateReturn.map" file. The CRMCreate schema, containing CRMCreate as its global element, is used as the input object.
CRM Connection
Pattern parameter Default Description
CRM organization URL

This pattern parameter defines the organization service URL of the CRM system. It must be a full URL to the organization service. For example, accessing CRM On-premise, http://ipaddress:port/orgName/XRMServices/2011/Organization.svc; accessing CRM Online, https://orgName.crm.dynamics.com/XRMServices/2011/Organization.svc.
CRM user name

This pattern parameter describes the CRM user name you used in CRM system.
CRM user password

This pattern parameter defines the password for the CRM user you specified.
CRM domain

This pattern parameter defines the CRM domain you used.
Logging
Pattern parameter Default Description
Logging on Cleared This pattern parameter determines whether the pattern instance includes the subflow for logging.

If this pattern parameter is cleared, the logging subflow is not included.

If this pattern parameter is selected, you must set values for Log queue manager and Log queue.
Log queue manager This pattern parameter defines the queue manager that is used for logging. You can leave this parameter blank if the broker queue manager is used for logging.

This pattern parameter is required only if Logging required is selected.
Log queue This pattern parameter defines the queue that is used for logging.

This pattern parameter is required only if Logging required is selected.
Error handling
Pattern parameter Default Description
Error logging on Selected This pattern parameter determines whether the pattern instance includes code for placing error messages on a queue.

If this pattern parameter is selected, values must be set for Error queue manager and Error queue.
Error queue manager This pattern parameter defines the queue manager that is used for error logging.

This pattern parameter is required only if Error message required is selected.
Error queue This pattern parameter defines the queue that is used for error messages.

This pattern parameter is required only if Error message required is selected.

All the pattern parameters are listed with their default values. Where no default is indicated, the default value is an empty string.

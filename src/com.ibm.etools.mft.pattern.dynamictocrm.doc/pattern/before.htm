Tasks to complete before applying the Microsoft Dynamics CRM Account Entity output: Dynamic transport input pattern

Before applying the pattern to produce a pattern instance, complete the following tasks:

  • Ensure that this pattern meets your business requirements and can be supported by both the requester and provider applications.
  • Choose a name for the pattern instance. Identify the purpose of the pattern application so that you can choose an appropriate name.
  • Decide which transport you want to use, and prepare the transport detail configurations. For example, create the required message queues for WebSphere MQ transport, create input file directories for file transport, or choose the URL suffix for HTTP transport.
  • Prepare input and output schemas for each operation.
  • Prepare the required Microsoft Dynamics CRM system connection information. This includes:
    • CRMDomain
    • CRMURI
    • CRMUser
    • CRMUser password
  • Decide if you require logging and error messages. If you require either of these, you must decide which queue manager and queue the messages will be sent to. You can send messages to a general queue that is used by multiple message flows, or you can use a queue specifically for the pattern instance.
